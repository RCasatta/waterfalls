Make a REST API request to get the entire wallet history in one shot with the Waterfalls endpoint.

Waterfalls request must contain only the "bitcoin part" of a ct descriptor, in other words the confidential descriptor without the blinding key.

In order to make REST API requests to the Blockstream Explorer Enterprise endpoints you first need to request an access token using your client_id and client_secret.

Note that the access token is the value of the access_token key in the JSON object returned from the token endpoint.

You will use these access tokens in the Authorization section of your REST API requests (Bearer type)

These access tokens expire after 300s (5 min) so you will have to refresh them periodically.

Endpoints:
Liquid mainnet: https://enterprise.blockstream.info/liquid/api
Liquid testnet: https://enterprise.blockstream.info/liquidtestnet/api
